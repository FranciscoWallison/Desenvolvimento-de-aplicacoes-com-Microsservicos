Aqui está a tradução dos termos e definições do certificado, mantendo a terminologia técnica padrão utilizada na área de cibersegurança e TI em português.

**A**

* **Absolute file path (Caminho absoluto do arquivo):** O caminho completo do arquivo, que começa a partir da raiz (root).
* **Access controls (Controles de acesso):** Controles de segurança que gerenciam o acesso, autorização e responsabilidade (accountability) da informação.
* **Active packet sniffing (Sniffing de pacotes ativo):** Um tipo de ataque onde os pacotes de dados são manipulados em trânsito.
* **Address Resolution Protocol (ARP):** Um protocolo de rede usado para determinar o endereço MAC do próximo roteador ou dispositivo no caminho.
* **Advanced persistent threat (APT - Ameaça Persistente Avançada):** Uma instância em que um agente de ameaça mantém acesso não autorizado a um sistema por um longo período de tempo.
* **Adversarial artificial intelligence (IA Adversária):** Uma técnica que manipula a tecnologia de inteligência artificial (IA) e aprendizado de máquina (ML) para conduzir ataques de forma mais eficiente.
* **Adware:** Um tipo de software legítimo que às vezes é usado para exibir anúncios digitais em aplicativos.
* **Algorithm (Algoritmo):** Um conjunto de regras usadas para resolver um problema.
* **Analysis (Análise):** A investigação e validação de alertas.
* **Angler phishing:** Uma técnica onde atacantes se passam por representantes de atendimento ao cliente nas redes sociais.
* **Anomaly-based analysis (Análise baseada em anomalias):** Um método de detecção que identifica comportamento anormal.
* **Antivirus software (Software antivírus):** Um programa de software usado para prevenir, detectar e eliminar malware e vírus.
* **Application (Aplicação/Aplicativo):** Um programa que executa uma tarefa específica.
* **Application programming interface (API) token (Token de API):** Um pequeno bloco de código criptografado que contém informações sobre um usuário.
* **Argument (Linux) (Argumento - Linux):** Informação específica necessária para um comando.
* **Argument (Python) (Argumento - Python):** Os dados trazidos para dentro de uma função quando ela é chamada.
* **Array (Matriz/Vetor):** Um tipo de dado que armazena dados em uma lista ordenada separada por vírgulas.
* **Artificial intelligence (AI) (Inteligência Artificial - IA):** Programas de computador que podem completar tarefas cognitivas tipicamente associadas à inteligência humana.
* **AI tool (Ferramenta de IA):** Software alimentado por IA que pode automatizar ou ajudar usuários com uma variedade de tarefas.
* **AI user (Usuário de IA):** Alguém que utiliza IA para completar uma tarefa pessoal ou profissional.
* **Assess (Avaliar):** O quinto passo do RMF do NIST que significa determinar se os controles estabelecidos estão implementados corretamente.
* **Asset (Ativo):** Um item percebido como tendo valor para uma organização.
* **Asset classification (Classificação de ativos):** A prática de rotular ativos com base na sensibilidade e importância para uma organização.
* **Asset inventory (Inventário de ativos):** Um catálogo de ativos que precisam ser protegidos.
* **Asset management (Gestão de ativos):** O processo de rastrear ativos e os riscos que os afetam.
* **Asymmetric encryption (Criptografia assimétrica):** O uso de um par de chaves pública e privada para criptografia e descriptografia de dados.

* **Attack surface (Superfície de ataque):** Todas as vulnerabilidades potenciais que um agente de ameaça poderia explorar.
* **Attack tree (Árvore de ataque):** Um diagrama que mapeia ameaças aos ativos.
* **Attack vectors (Vetores de ataque):** Os caminhos que os atacantes usam para penetrar as defesas de segurança.
* **Authentication (Autenticação):** O processo de verificar quem alguém é.
* **Authorization (Autorização):** O conceito de conceder acesso a recursos específicos em um sistema.
* **Authorize (Autorizar):** O sexto passo do RMF do NIST que se refere a ser responsável pelos riscos de segurança e privacidade que podem existir em uma organização.
* **Automation (Automação):** O uso de tecnologia para reduzir o esforço humano e manual para realizar tarefas comuns e repetitivas.
* **Availability (Disponibilidade):** A ideia de que os dados estão acessíveis para aqueles que estão autorizados a acessá-los.

**B**

* **Baiting (Isca):** Uma tática de engenharia social que tenta as pessoas a comprometerem sua segurança (ex: oferecendo algo em troca).
* **Bandwidth (Largura de banda):** A capacidade máxima de transmissão de dados através de uma rede, medida em bits por segundo.
* **Baseline configuration (Configuração de linha de base):** Um conjunto documentado de especificações dentro de um sistema que é usado como base para futuras compilações, lançamentos e atualizações.
* **Bash:** O shell padrão na maioria das distribuições Linux.
* **Basic auth (Autenticação básica):** A tecnologia usada para estabelecer a solicitação de um usuário para acessar um servidor.
* **Basic Input/Output System (BIOS):** Um microchip que contém instruções de carregamento para o computador e é prevalente em sistemas mais antigos.
* **Behavioral interview (Entrevista comportamental):** Um tipo de entrevista projetada para entender como você respondeu em situações passadas para prever seu desempenho futuro.
* **Biometrics (Biometria):** As características físicas únicas que podem ser usadas para verificar a identidade de uma pessoa.
* **Bit:** A menor unidade de medida de dados em um computador.
* **Boolean data (Dados booleanos):** Dados que podem ter apenas um de dois valores: Verdadeiro (True) ou Falso (False).
* **Bootloader:** Um programa de software que inicializa o sistema operacional.
* **Botnet:** Uma coleção de computadores infectados por malware que estão sob o controle de um único agente de ameaça, conhecido como "bot-herder" (pastor de bots).
* **Bracket notation (Notação de colchetes):** Os índices colocados entre colchetes.
* **Broken chain of custody (Quebra da cadeia de custódia):** Inconsistências na coleta e registro de evidências na cadeia de custódia.
* **Brute force attack (Ataque de força bruta):** O processo de tentativa e erro para descobrir informações privadas (como senhas).
* **Bug bounty (Recompensa por bugs):** Programas que incentivam hackers freelancers a encontrar e relatar vulnerabilidades.
* **Built-in function (Função embutida/nativa):** Uma função que existe dentro do Python e pode ser chamada diretamente.
* **Business continuity (Continuidade de negócios):** A capacidade de uma organização de manter sua produtividade diária estabelecendo planos de recuperação de desastres e riscos.
* **Business continuity plan (BCP - Plano de continuidade de negócios):** Um documento que descreve os procedimentos para sustentar as operações de negócios durante e após uma interrupção significativa.
* **Business Email Compromise (BEC - Comprometimento de E-mail Corporativo):** Um tipo de ataque de phishing onde um agente de ameaça se passa por uma fonte conhecida para obter vantagem financeira.

**C**

* **Career Dreamer:** Uma ferramenta experimental que usa IA para tornar a exploração de carreira mais fácil e personalizada.
* **Career identity statement (Declaração de identidade de carreira):** Algumas frases sobre você que fornecem uma visão geral rápida, mas perspicaz, de suas principais habilidades, interesses, objetivos de carreira e o que você ama fazer.
* **Case interview (Entrevista de caso):** Um tipo de entrevista projetada para avaliar suas habilidades de resolução de problemas, análise e comunicação, apresentando desafios de negócios do mundo real para você analisar e resolver.
* **Categorize (Categorizar):** O segundo passo do RMF do NIST que é usado para desenvolver processos e tarefas de gerenciamento de risco.
* **CentOS:** Uma distribuição de código aberto (open-source) que é intimamente relacionada ao Red Hat.
* **Central Processing Unit (CPU - Unidade Central de Processamento):** O processador principal de um computador, usado para realizar tarefas gerais de computação.
* **Chain of custody (Cadeia de custódia):** O processo de documentar a posse e o controle de evidências durante o ciclo de vida de um incidente.
* **Chat:** Uma caixa de entrada em uma ferramenta de IA onde você pode adicionar prompts.
* **Chronicle (Google SecOps):** Uma ferramenta nativa da nuvem projetada para reter, analisar e pesquisar dados.
* **Cipher (Cifra):** Um algoritmo que criptografa informações.
* **Cloud-based firewalls (Firewalls baseados em nuvem):** Firewalls de software hospedados pelo provedor de serviços em nuvem.
* **Cloud computing (Computação em nuvem):** A prática de usar servidores remotos, aplicativos e serviços de rede hospedados na internet em vez de em dispositivos físicos locais.
* **Cloud network (Rede em nuvem):** Uma coleção de servidores ou computadores que armazena recursos e dados em data centers remotos que podem ser acessados via internet.
* **Cloud security (Segurança na nuvem):** O processo de garantir que os ativos armazenados na nuvem estejam configurados corretamente e que o acesso a esses ativos seja limitado a usuários autorizados.
* **Command (Comando):** Uma instrução dizendo ao computador para fazer algo.
* **Command and control (C2 - Comando e Controle):** As técnicas usadas por agentes maliciosos para manter comunicações com sistemas comprometidos.
* **Command-line interface (CLI - Interface de linha de comando):** Uma interface de usuário baseada em texto que usa comandos para interagir com o computador.
* **Comment (Comentário):** Uma nota que os programadores fazem sobre a intenção por trás de seu código.
* **Common Event Format (CEF):** Um formato de log que usa pares chave-valor para estruturar dados e identificar campos e seus valores correspondentes.
* **Common Vulnerabilities and Exposures (CVE®) list (Lista de Vulnerabilidades e Exposições Comuns):** Um dicionário acessível publicamente de vulnerabilidades e exposições conhecidas.
* **Common Vulnerability Scoring System (CVSS):** Um sistema de medição que pontua a gravidade de uma vulnerabilidade.
* **Compliance (Conformidade):** O processo de aderir a padrões internos e regulamentos externos.
* **Computer security incident response teams (CSIRT):** Um grupo especializado de profissionais de segurança treinados em gerenciamento e resposta a incidentes.
* **Computer virus (Vírus de computador):** Código malicioso escrito para interferir nas operações do computador e causar danos a dados e software.
* **Conditional statement (Declaração condicional):** Uma declaração que avalia o código para determinar se ele atende a um conjunto especificado de condições.
* **Confidential data (Dados confidenciais):** Dados que geralmente têm limites quanto ao número de pessoas que têm acesso a eles.
* **Confidentiality (Confidencialidade):** A ideia de que apenas usuários autorizados podem acessar ativos ou dados específicos.
* **Confidentiality, integrity, availability (CIA) triad (Tríade CIA):** Um modelo que ajuda a informar como as organizações consideram o risco ao configurar sistemas e políticas de segurança (Confidencialidade, Integridade, Disponibilidade).
![alt text](image-2.png)
* **Configuration file (Arquivo de configuração):** Um arquivo usado para configurar as definições de um aplicativo.
* **Containment (Contenção):** O ato de limitar e prevenir danos adicionais causados por um incidente.
* **Controlled zone (Zona controlada):** Uma sub-rede que protege a rede interna da zona não controlada.
* **Cover letter (Carta de apresentação):** Um documento personalizado que mostra suas habilidades de comunicação e personalidade, demonstrando como suas habilidades e experiência se alinham com as necessidades do empregador.
* **Cross-site scripting (XSS):** Um ataque de injeção que insere código em um site ou aplicativo web vulnerável.
* **Crowdsourcing:** A prática de coletar informações usando a entrada e colaboração do público.
* **Cryptographic attack (Ataque criptográfico):** Um ataque que afeta formas seguras de comunicação entre um remetente e o destinatário pretendido.
* **Cryptographic key (Chave criptográfica):** Um mecanismo que descriptografa texto cifrado.
* **Cryptography (Criptografia):** O processo de transformar informações em uma forma que leitores não intencionais não consigam entender.
* **Cryptojacking:** Uma forma de malware que instala software para minerar criptomoedas ilegalmente.
* **CVE Numbering Authority (CNA):** Uma organização que se voluntaria para analisar e distribuir informações sobre CVEs elegíveis.
* **Cybersecurity (Cibersegurança/Segurança Cibernética):** A prática de garantir a confidencialidade, integridade e disponibilidade da informação protegendo redes, dispositivos, pessoas e dados contra acesso não autorizado ou exploração criminosa.

**D**

* **Data (Dados):** Informação que é traduzida, processada ou armazenada por um computador.
* **Data at rest (Dados em repouso):** Dados que não estão sendo acessados atualmente.
* **Data controller (Controlador de dados):** Uma pessoa que determina o procedimento e o propósito para o processamento de dados.
* **Data custodian (Custodiante de dados):** Qualquer pessoa ou coisa responsável pelo manuseio, transporte e armazenamento seguro de informações.
* **Data exfiltration (Exfiltração de dados):** Transmissão não autorizada de dados de um sistema.
* **Data in transit (Dados em trânsito):** Dados viajando de um ponto a outro.
* **Data in use (Dados em uso):** Dados sendo acessados por um ou mais usuários.
* **Data owner (Proprietário dos dados):** A pessoa que decide quem pode acessar, editar, usar ou destruir suas informações.
* **Data packet (Pacote de dados):** Uma unidade básica de informação que viaja de um dispositivo para outro dentro de uma rede.
* **Data point (Ponto de dados):** Um pedaço específico de informação.
* **Data processor (Processador de dados):** Uma pessoa responsável por processar dados em nome do controlador de dados.
* **Data protection officer (DPO - Encarregado de proteção de dados):** Um indivíduo responsável por monitorar a conformidade dos procedimentos de proteção de dados de uma organização.
* **Data type (Tipo de dados):** Uma categoria para um tipo específico de item de dados.
* **Database (Banco de dados):** Uma coleção organizada de informações ou dados.
* **Date and time data (Dados de data e hora):** Dados representando uma data e/ou hora.
* **Debugger (Depurador):** Uma ferramenta de software que ajuda a localizar a fonte de um erro e avaliar suas causas.
* **Debugging (Depuração):** A prática de identificar e corrigir erros no código.
* **Defense in depth (Defesa em profundidade):** Uma abordagem em camadas para o gerenciamento de vulnerabilidades que reduz o risco.
* **Denial of service (DoS) attack (Ataque de negação de serviço):** Um ataque que visa uma rede ou servidor e o inunda com tráfego de rede.
* **Detect (Detectar):** Uma função principal do NIST relacionada à identificação de potenciais incidentes de segurança e melhoria das capacidades de monitoramento para aumentar a velocidade e eficiência das detecções.
* **Detection (Detecção):** A descoberta imediata de eventos de segurança.
* **Dictionary data (Dados de dicionário):** Dados que consistem em um ou mais pares chave-valor.
* **Digital certificate (Certificado digital):** Um arquivo que verifica a identidade de um titular de chave pública.
* **Digital forensics (Forense digital):** A prática de coletar e analisar dados para determinar o que aconteceu após um ataque.
* **Directory (Diretório/Pasta):** Um arquivo que organiza onde outros arquivos são armazenados.
* **Disaster recovery plan (Plano de recuperação de desastres):** Um plano que permite que a equipe de segurança de uma organização descreva as etapas necessárias para minimizar o impacto de um incidente de segurança.
* **Distributed denial of service (DDoS) attack (Ataque de negação de serviço distribuído):** Um tipo de ataque de negação de serviço que usa vários dispositivos ou servidores localizados em diferentes locais para inundar a rede alvo com tráfego indesejado.
* **Distributions (Distribuições/Distros):** As diferentes versões do Linux.
* **Documentation (Documentação):** Qualquer forma de conteúdo registrado que é usado para um propósito específico.
* **DOM-based XSS attack (Ataque XSS baseado em DOM):** Uma instância em que o script malicioso existe na página da web que um navegador carrega.
* **Domain Name System (DNS - Sistema de Nomes de Domínio):** Um protocolo de rede que traduz nomes de domínio da internet em endereços IP.
* **Dropper:** Um tipo de malware que vem carregado com código malicioso, o qual é entregue e instalado em um sistema alvo.

**E**

* **Elevator pitch (Discurso de elevador):** Um breve resumo de sua experiência, habilidades e antecedentes.
* **Encapsulation (Encapsulamento):** Um processo realizado por um serviço VPN que protege seus dados envolvendo dados sensíveis em outros pacotes de dados.
* **Encryption (Criptografia):** O processo de converter dados de um formato legível para um formato codificado.
* **Endpoint (Ponto de extremidade):** Qualquer dispositivo conectado a uma rede.
* **Endpoint detection and response (EDR - Detecção e resposta de endpoint):** Um aplicativo que monitora um endpoint em busca de atividade maliciosa.
* **Eradication (Erradicação):** A remoção completa dos elementos do incidente de todos os sistemas afetados.
* **Escalation policy (Política de escalonamento):** Um conjunto de ações que descrevem quem deve ser notificado quando ocorre um alerta de incidente e como esse incidente deve ser tratado.
* **Event (Evento):** Uma ocorrência observável em uma rede, sistema ou dispositivo.
* **Exception (Exceção):** Um erro que envolve código que não pode ser executado, embora esteja sintaticamente correto.
* **Exclusive operator (Operador exclusivo):** Um operador que não inclui o valor de comparação.
* **Exploit (Exploração):** Uma maneira de tirar vantagem de uma vulnerabilidade.
* **Exposure (Exposição):** Um erro que pode ser explorado por uma ameaça.
* **External threat (Ameaça externa):** Qualquer coisa fora da organização que tenha o potencial de prejudicar os ativos organizacionais.

**F**

* **False negative (Falso negativo):** Um estado em que a presença de uma ameaça não é detectada.
* **False positive (Falso positivo):** Um alerta que detecta incorretamente a presença de uma ameaça.
* **Fileless malware (Malware sem arquivo):** Malware que não precisa ser instalado pelo usuário porque usa programas legítimos que já estão instalados para infectar um computador.
* **File path (Caminho do arquivo):** A localização de um arquivo ou diretório.
* **Filesystem Hierarchy Standard (FHS):** O componente do sistema operacional Linux que organiza os dados.
* **Filtering (Filtragem):** Seleção de dados que correspondem a uma determinada condição.
* **Final report (Relatório final):** Documentação que fornece uma revisão abrangente de um incidente.
* **Firewall:** Um dispositivo de segurança de rede que monitora o tráfego para ou de uma rede.
* **Float data (Dados de ponto flutuante):** Dados consistindo de um número com um ponto decimal.
* **Foreign key (Chave estrangeira):** Uma coluna em uma tabela que é uma chave primária em outra tabela.
* **Forward proxy server (Servidor proxy de encaminhamento):** Um servidor que regula e restringe o acesso de uma pessoa à internet.
* **Function (Função):** Uma seção de código que pode ser reutilizada em um programa.

**G**

* **Gemini:** Um modelo de IA generativa que pode entender e combinar vários tipos de informações, incluindo texto, código, áudio, imagens e vídeo.
* **Gemini Deep Research:** Um recurso dentro dos aplicativos Gemini que permite aos usuários realizar pesquisas aprofundadas navegando em vários sites, analisando as informações e gerando relatórios de várias páginas.
* **Gemini Live:** Um recurso no aplicativo Gemini que permite conversas de voz naturais e em tempo real.
* **Generative AI (IA Generativa):** IA que pode gerar novo conteúdo, como texto, imagens ou outras mídias.
* **Global variable (Variável global):** Uma variável que está disponível em todo o programa.
* **Graphical user interface (GUI - Interface gráfica do usuário):** Uma interface de usuário que usa ícones na tela para gerenciar diferentes tarefas no computador.
* **Growth mindset (Mentalidade de crescimento):** Um compromisso com o aprendizado e desenvolvimento contínuos.

**H**

* **Hacker:** Qualquer pessoa que usa computadores para obter acesso a sistemas de computador, redes ou dados.
* **Hacktivist (Hacktivista):** Uma pessoa que usa hacking para alcançar um objetivo político.
* **Hard drive (Disco rígido):** Um componente de hardware usado para memória de longo prazo.
* **Hardware:** Os componentes físicos de um computador.
* **Hash collision (Colisão de hash):** Uma instância em que diferentes entradas produzem o mesmo valor de hash.
* **Hash function (Função hash):** Um algoritmo que produz um código que não pode ser descriptografado.
* **Hash table (Tabela hash):** Uma estrutura de dados usada para armazenar e referenciar valores hash.
* **Health Insurance Portability and Accountability Act (HIPAA):** Uma lei federal dos EUA estabelecida para proteger as informações de saúde dos pacientes.
* **Honeypot:** Um sistema ou recurso criado como isca vulnerável a ataques com o objetivo de atrair potenciais intrusos.
* **Host-based intrusion detection system (HIDS - Sistema de detecção de intrusão baseado em host):** Um aplicativo que monitora a atividade do host no qual está instalado.
* **Hub:** Um dispositivo de rede que transmite informações para todos os dispositivos na rede.
* **Human touch (Toque humano):** O ato de personalizar ou refinar a saída da IA para garantir que a personalidade, o tom, a voz, o uso da linguagem e a precisão sejam tão autênticos quanto possível.
* **Hypertext Transfer Protocol (HTTP):** Um protocolo de camada de aplicação que fornece um método de comunicação entre clientes e servidores de sites.
* **Hypertext Transfer Protocol Secure (HTTPS):** Um protocolo de rede que fornece um método seguro de comunicação entre clientes e servidores de sites.

**I**

* **Identify (Identificar):** Uma função principal do NIST relacionada ao gerenciamento de risco de segurança cibernética e seu efeito nas pessoas e ativos de uma organização.
* **Identity and access management (IAM - Gerenciamento de identidade e acesso):** Uma coleção de processos e tecnologias que ajuda as organizações a gerenciar identidades digitais em seu ambiente.
* **IEEE 802.11 (Wi-Fi):** Um conjunto de padrões que definem a comunicação para LANs sem fio.
* **Immutable (Imutável):** Um objeto que não pode ser alterado após ser criado e ter um valor atribuído.
* **Implement (Implementar):** O quarto passo do RMF do NIST que significa implementar planos de segurança e privacidade para uma organização.
* **Improper usage (Uso indevido):** Um tipo de incidente que ocorre quando um funcionário de uma organização viola as políticas de uso aceitável da organização.
* **Incident (Incidente):** Uma ocorrência que realmente ou iminentemente coloca em risco, sem autoridade legal, a confidencialidade, integridade ou disponibilidade de informações ou de um sistema de informação; ou constitui uma violação ou ameaça iminente de violação da lei, políticas de segurança, procedimentos de segurança ou políticas de uso aceitável.
* **Incident escalation (Escalonamento de incidente):** O processo de identificar um potencial incidente de segurança, triá-lo e passá-lo para um membro da equipe mais experiente.
* **Incident handler’s journal (Diário do manipulador de incidentes):** Uma forma de documentação usada na resposta a incidentes.
* **Incident response (Resposta a incidentes):** A tentativa rápida de uma organização de identificar um ataque, conter o dano e corrigir os efeitos de uma violação de segurança.
* **Incident response plan (Plano de resposta a incidentes):** Um documento que descreve os procedimentos a serem seguidos em cada etapa da resposta a incidentes.
* **Inclusive operator (Operador inclusivo):** Um operador que inclui o valor de comparação.
* **Indentation (Indentação):** Espaço adicionado no início de uma linha de código.
* **Index (Índice):** Um número atribuído a cada elemento em uma sequência que indica sua posição.
* **Indicators of attack (IoA - Indicadores de ataque):** A série de eventos observados que indicam um incidente em tempo real.
* **Indicators of compromise (IoC - Indicadores de compromisso):** Evidência observável que sugere sinais de um potencial incidente de segurança.
* **Information privacy (Privacidade da informação):** A proteção contra acesso e distribuição não autorizados de dados.
* **Information security (InfoSec - Segurança da informação):** A prática de manter dados em todos os estados longe de usuários não autorizados.
* **Injection attack (Ataque de injeção):** Código malicioso inserido em um aplicativo vulnerável.
* **Input (Entrada):** A linguagem que você fornece a um assistente de IA; tipicamente uma solicitação ou prompt para gerar algo específico.
* **Input validation (Validação de entrada):** Programação que valida entradas de usuários e outros programas.
* **Integer data (Dados inteiros):** Dados consistindo de um número que não inclui um ponto decimal.
* **Integrated development environment (IDE - Ambiente de desenvolvimento integrado):** Um aplicativo de software para escrever código que fornece assistência de edição e ferramentas de correção de erros.
* **Integrity (Integridade):** A ideia de que os dados estão corretos, autênticos e confiáveis.
* **Internal hardware (Hardware interno):** Os componentes necessários para executar o computador.
* **Internal threat (Ameaça interna):** Um funcionário atual ou antigo, fornecedor externo ou parceiro de confiança que representa um risco de segurança.

* **Speed (Velocidade):** A taxa na qual um dispositivo envia e recebe dados, medida em bits por segundo.
* **Splunk Cloud:** Uma ferramenta hospedada na nuvem usada para coletar, pesquisar e monitorar dados de log.
* **Splunk Enterprise:** Uma ferramenta auto-hospedada usada para reter, analisar e pesquisar dados de log de uma organização.
* **Spyware:** Malware usado para coletar e vender informações sem consentimento.
* **SQL (Structured Query Language - Linguagem de Consulta Estruturada):** Uma linguagem de programação usada para criar, interagir e solicitar informações de um banco de dados.
* **SQL injection (Injeção de SQL):** Um ataque que executa consultas inesperadas em um banco de dados.
* **Stakeholder (Parte interessada):** Um indivíduo ou grupo que tem interesse em qualquer decisão ou atividade de uma organização.
* **Standard error (Erro padrão):** Uma mensagem de erro retornada pelo SO através do shell.
* **Standard input (Entrada padrão):** Informação recebida pelo SO via linha de comando.
* **Standard output (Saída padrão):** Informação retornada pelo SO através do shell.
* **Standards (Padrões):** Referências que informam como definir políticas.
* **STAR method (Método STAR):** Uma estrutura de narrativa que ajuda você a comunicar suas experiências e habilidades (Situação, Tarefa, Ação e Resultado).
* **Stateful:** Uma classe de firewall que acompanha as informações que passam por ele e filtra proativamente ameaças.
* **Stateless:** Uma classe de firewall que opera com base em regras predefinidas e que não acompanha informações de pacotes de dados.
* **Stored XSS attack (Ataque XSS armazenado):** Uma instância em que o script malicioso é injetado diretamente no servidor.
* **String concatenation (Concatenação de string):** O processo de juntar duas strings.
* **String data (Dados de string):** Dados consistindo de uma sequência ordenada de caracteres.
* **Style guide (Guia de estilo):** Um manual que informa a escrita, formatação e design de documentos.
* **Subnetting (Sub-redes):** A subdivisão de uma rede em grupos lógicos chamados sub-redes.
* **Substring:** Uma sequência contínua de caracteres dentro de uma string.
* **Sudo:** Um comando que concede temporariamente permissões elevadas a usuários específicos.
* **Supply-chain attack (Ataque à cadeia de suprimentos):** Um ataque que visa sistemas, aplicativos, hardware e/ou software para localizar uma vulnerabilidade onde malware pode ser implantado.
* **Suricata:** Um sistema de detecção de intrusão, sistema de prevenção de intrusão e ferramenta de análise de rede de código aberto.
* **Switch:** Um dispositivo que faz conexões entre dispositivos específicos em uma rede enviando e recebendo dados entre eles.
* **Symmetric encryption (Criptografia simétrica):** O uso de uma única chave secreta para trocar informações.
* **Synchronize (SYN) flood attack (Ataque de inundação SYN):** Um tipo de ataque DoS que simula uma conexão TCP/IP e inunda um servidor com pacotes SYN.
* **Syntax (Sintaxe):** As regras que determinam o que é corretamente estruturado em uma linguagem de computação.
* **Syntax error (Erro de sintaxe):** Um erro que envolve o uso inválido de uma linguagem de programação.

**T**

* **Tailgating:** Uma tática de engenharia social na qual pessoas não autorizadas seguem uma pessoa autorizada para uma área restrita.
* **TCP/IP model (Modelo TCP/IP):** Uma estrutura usada para visualizar como os dados são organizados e transmitidos através de uma rede.
* **tcpdump:** Um analisador de protocolo de rede de linha de comando.
* **Technical interview (Entrevista técnica):** Um tipo de entrevista projetada para avaliar sua competência técnica.
* **Technical skills (Habilidades técnicas):** Habilidades que requerem conhecimento de ferramentas, procedimentos e políticas específicas.
* **Telemetry (Telemetria):** A coleta e transmissão de dados para análise.
* **Threat (Ameaça):** Qualquer circunstância ou evento que possa impactar negativamente os ativos.
* **Threat actor (Agente de ameaça):** Qualquer pessoa ou grupo que apresenta um risco de segurança.
* **Threat hunting (Caça a ameaças):** A busca proativa por ameaças em uma rede.
* **Threat intelligence (Inteligência de ameaças):** Informações sobre ameaças baseadas em evidências que fornecem contexto sobre ameaças existentes ou emergentes.
* **Threat modeling (Modelagem de ameaças):** O processo de identificar ativos, suas vulnerabilidades e como cada um está exposto a ameaças.
* **Transferable skills (Habilidades transferíveis):** Habilidades que você aprendeu em uma situação que podem ser aplicadas a uma situação completamente diferente.
* **Transmission Control Protocol (TCP):** Um protocolo de comunicação da internet que permite que dois dispositivos formem uma conexão e transmitam dados.
* **Triage (Triagem):** A priorização de incidentes de acordo com seu nível de importância ou urgência.
* **Trojan horse (Cavalo de Troia):** Malware que parece um arquivo ou programa legítimo.
* **True negative (Verdadeiro negativo):** Um estado onde não há detecção de atividade maliciosa.
* **True positive (Verdadeiro positivo):** Um alerta que detecta corretamente a presença de um ataque.
* **Tuple data (Dados de tupla):** Estrutura de dados que consiste em uma coleção de dados que não pode ser alterada.
* **Type error (Erro de tipo):** Um erro que resulta do uso do tipo de dados errado.

**U**

* **Ubuntu:** Uma distribuição de código aberto e amigável ao usuário que é amplamente utilizada em segurança e outras indústrias.
* **Unauthorized access (Acesso não autorizado):** Um tipo de incidente que ocorre quando um indivíduo obtém acesso digital ou físico a um sistema ou aplicativo sem permissão.
* **Uncontrolled zone (Zona não controlada):** Qualquer rede fora do controle da sua organização.
* **Unified Extensible Firmware Interface (UEFI):** Um microchip que contém instruções de carregamento para o computador e substitui o BIOS em sistemas mais modernos.
* **USB baiting (Isca USB):** Um ataque no qual um agente de ameaça estrategicamente deixa um pendrive USB com malware para um funcionário encontrar e instalar, infectando a rede sem saber.
* **User (Usuário):** A pessoa interagindo com um computador.
* **User Datagram Protocol (UDP):** Um protocolo sem conexão que não estabelece uma conexão entre dispositivos antes das transmissões.
* **User-defined function (Função definida pelo usuário):** Uma função que os programadores projetam para suas necessidades específicas.
* **User interface (Interface de usuário):** Um programa que permite ao usuário controlar as funções do sistema operacional.
* **User provisioning (Provisionamento de usuário):** O processo de criar e manter a identidade digital de um usuário.

**V**

* **Variable (Variável):** Um contêiner que armazena dados.
* **Virtual machine (VM - Máquina virtual):** Uma versão virtual de um computador físico.
* **Virtual Private Network (VPN - Rede Privada Virtual):** Um serviço de segurança de rede que altera seu endereço IP público e oculta sua localização virtual para que você possa manter seus dados privados ao usar uma rede pública.
* **Virus (Vírus):** Código malicioso escrito para interferir nas operações do computador e causar danos a dados e software.
* **VirusTotal:** Um serviço que permite a qualquer pessoa analisar arquivos suspeitos, domínios, URLs e endereços IP em busca de conteúdo malicioso.
* **Vishing:** A exploração da comunicação de voz eletrônica para obter informações sensíveis ou se passar por uma fonte conhecida.
* **Visual dashboard (Painel visual/Dashboard):** Uma maneira de exibir vários tipos de dados rapidamente em um só lugar.
* **Vulnerability (Vulnerabilidade):** Uma fraqueza que pode ser explorada por uma ameaça.
* **Vulnerability assessment (Avaliação de vulnerabilidade):** O processo de revisão interna dos sistemas de segurança de uma organização.
* **Vulnerability management (Gerenciamento de vulnerabilidade):** O processo de encontrar e corrigir vulnerabilidades.
* **Vulnerability scanner (Scanner de vulnerabilidade):** Software que compara automaticamente vulnerabilidades e exposições comuns existentes com as tecnologias na rede.

**W**

* **Watering hole attack:** Um tipo de ataque quando um agente de ameaça compromete um site frequentemente visitado por um grupo específico de usuários.
* **Web-based exploits (Explorações baseadas na Web):** Código ou comportamento malicioso usado para tirar vantagem de falhas de codificação em um aplicativo web.
* **Whaling:** Uma categoria de tentativas de spear phishing que visam executivos de alto escalão em uma organização.
* **Wide Area Network (WAN - Rede de longa distância):** Uma rede que abrange uma grande área geográfica como uma cidade, estado ou país.
* **Wi-Fi Protected Access (WPA):** Um protocolo de segurança sem fio para dispositivos se conectarem à internet.
* **Wildcard (Curinga):** Um caractere especial que pode ser substituído por qualquer outro caractere.
* **Wireshark:** Um analisador de protocolo de rede de código aberto.
* **World-writable file (Arquivo gravável por todos):** Um arquivo que pode ser alterado por qualquer pessoa no mundo.
* **Worm (Verme):** Malware que pode se duplicar e se espalhar pelos sistemas por conta própria.

**X**

* **X-Y-Z approach (Abordagem X-Y-Z):** Uma fórmula ou técnica que ajuda você a estruturar suas conquistas para mostrar claramente seu impacto.
* **YARA-L:** Uma linguagem de computador usada para criar regras para pesquisar dados de log ingeridos.
* **Zero-day (Dia zero):** Uma exploração (exploit) que era anteriormente desconhecida.
