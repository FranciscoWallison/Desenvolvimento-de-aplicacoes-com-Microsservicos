## 📖 O que é um Cookie?

> **Cookies** são **pequenos arquivos de texto** criados pelos sites visitados e armazenados no computador do usuário, através do navegador.

Apesar do nome doce, são apenas pares **chave=valor** persistidos pelo navegador. Toda vez que o usuário volta ao mesmo site, o navegador **devolve** os cookies daquele domínio — é assim que o site "lembra" de você.

### O que pode estar armazenado

- 🔖 Páginas visitadas
- 👤 Dados fornecidos pelo usuário (nome, e-mail)
- 🛒 Itens no carrinho
- 🌍 Idioma e tema preferidos
- 🆔 ID de sessão (após login)
- 📊 Identificadores para rastreamento

### Como funcionam (fluxo simplificado)

```
1ª visita ─────────────────────────────────────────────
┌──────────┐  GET /                              ┌──────────┐
│ Browser  │ ──────────────────────────────────► │ Servidor │
└──────────┘ ◄────────────────────────────────── └──────────┘
              200 OK + Set-Cookie: id=abc123

Visitas seguintes ──────────────────────────────────────
┌──────────┐  GET /  Cookie: id=abc123          ┌──────────┐
│ Browser  │ ──────────────────────────────────► │ Servidor │
└──────────┘ ◄────────────────────────────────── └──────────┘
              "Bem-vindo de volta, Wallison!"
```

## 🗂️ Os 3 Tipos de Cookies

### 1️⃣ Cookies de Sessão (*Session Cookies*)

| Característica | Detalhe |
|----------------|---------|
| **Onde fica** | Memória temporária (RAM) do navegador |
| **Tempo de vida** | Até você **fechar o navegador** |
| **Uso típico** | Manter login ativo, carrinho durante a navegação |
| **Risco de privacidade** | Baixo (apagado automaticamente) |

**Exemplo real:**
> Você loga no Mercado Livre, navega entre páginas e adiciona itens ao carrinho. O cookie de sessão mantém você logado. Ao fechar o navegador, ele some.

---

### 2️⃣ Cookies Persistentes (*Persistent Cookies*)

| Característica | Detalhe |
|----------------|---------|
| **Onde fica** | Disco rígido (HD/SSD) |
| **Tempo de vida** | Até a **data de expiração** OU exclusão manual |
| **Uso típico** | "Lembrar de mim", anúncios direcionados, analytics |
| **Risco de privacidade** | Médio a alto (rastreamento entre sessões) |

**Exemplo real:**
> Você pesquisa "tênis de corrida" na Amazon. Nas próximas semanas, anúncios desses tênis aparecem em outros sites. Isso acontece graças a cookies persistentes de **terceiros** (third-party).

> 💡 Esses cookies são os principais responsáveis pelos anúncios "perseguidores" que vemos por toda a internet.

---

### 3️⃣ Cookies Maliciosos (*Malicious / Tracking Cookies*)

| Característica | Detalhe |
|----------------|---------|
| **O que faz** | Coleta dados de atividade sem permissão clara |
| **Como entra** | Via sites comprometidos, scripts de terceiros, anúncios maliciosos |
| **Risco** | Roubo de sessão, rastreamento massivo, espionagem |

**Cenários reais:**
- 🚨 **Session Hijacking** — atacante rouba o cookie de sessão e se faz passar por você
- 🚨 **Tracking abusivo** — perfilamento sem consentimento
- 🚨 **XSS exfiltrando cookies** — script malicioso lê e envia seus cookies para um servidor do atacante

> 📌 Por isso existem as flags `HttpOnly` (cookie não acessível via JavaScript) e `Secure` (só viaja em HTTPS) — ver [50.md](50.md) para detalhes técnicos.

## 📊 Comparativo Rápido

| Tipo | Duração | Onde mora | Risco |
|------|---------|-----------|-------|
| **Sessão** | Até fechar o navegador | RAM | 🟢 Baixo |
| **Persistente** | Até data de expiração | Disco | 🟡 Médio |
| **Malicioso** | Variável | Disco/RAM | 🔴 Alto |

### Outras classificações úteis

| Classificação | Diferença |
|---------------|-----------|
| **First-party** | Criado pelo próprio site visitado (`amazon.com` cria cookie em `amazon.com`) |
| **Third-party** | Criado por outro domínio embutido (anúncio do `doubleclick.net` em `nytimes.com`) — alvo principal das regulações de privacidade |
| **Zombie cookie** | Cookie que se "recria" após apagado (usa armazenamento alternativo como Flash, localStorage) |
| **Supercookie** | Cookie inserido pelo provedor de internet (ISP), difícil de remover |

## ⚖️ LGPD e Cookies — O Direito de Consentimento

A **Lei Geral de Proteção de Dados (LGPD — Lei nº 13.709/2018)** brasileira tem como foco:

> 🎯 **Proteger os usuários** quanto ao processamento e à livre circulação de seus dados pessoais.

### O que a LGPD exige sobre cookies

| Requisito | Detalhe |
|-----------|---------|
| **Consentimento expresso** | O usuário precisa **optar** por aceitar — não pode ser pré-marcado |
| **Granularidade** | Permitir aceitar/recusar **por tipo** (essenciais, analytics, marketing) |
| **Facilidade de uso** | Recusar deve ser tão fácil quanto aceitar |
| **Reversibilidade** | Usuário pode **mudar de ideia** a qualquer momento |
| **Transparência** | Política clara sobre quais dados são coletados e por quê |

### Comparação com regulações similares

| Lei | País/Região | Foco principal |
|-----|-------------|---------------|
| **LGPD** | 🇧🇷 Brasil | Proteção de dados pessoais |
| **GDPR** | 🇪🇺 União Europeia | Inspirou a LGPD; aplicação extraterritorial |
| **CCPA** | 🇺🇸 Califórnia | Direito de saber e excluir dados |
| **ePrivacy Directive** | 🇪🇺 União Europeia | Cookies especificamente |

### Como deve ser um banner de cookies CORRETO

```
┌────────────────────────────────────────────────────────┐
│ 🍪 Este site usa cookies                               │
│                                                        │
│ Usamos cookies para melhorar sua experiência.          │
│ Escolha o que deseja permitir:                         │
│                                                        │
│  [✓] Essenciais (necessários para o site funcionar)   │
│  [ ] Analytics (estatísticas de uso)                  │
│  [ ] Marketing (anúncios personalizados)              │
│                                                        │
│  [ Recusar todos ]  [ Aceitar selecionados ]  [ Aceitar todos ] │
└────────────────────────────────────────────────────────┘
```

### Como NÃO deve ser (anti-padrões comuns)

- ❌ Apenas botão "Aceitar" (sem opção de recusar)
- ❌ Botão "Aceitar" gigante verde + link minúsculo "Mais opções"
- ❌ Caixas pré-marcadas como aceitas
- ❌ Banner que volta a aparecer mesmo após recusar
- ❌ Bloquear o site se você recusar cookies não-essenciais

## 🔍 Como Ver os Cookies Armazenados no seu Navegador

### Google Chrome / Chromium / Edge

```
1. F12 (abrir DevTools)
2. Aba: Application → Storage → Cookies
3. Selecionar o domínio
```

Ou:
```
chrome://settings/cookies/detail?site=<dominio>
```

### Firefox

```
1. F12 → Storage (Armazenamento)
2. Cookies → selecionar domínio
```

Ou:
```
about:preferences#privacy → Gerenciar Dados
```

### Safari

```
Preferências → Privacidade → Gerenciar dados do site
```

> 💡 **Dica de pentest:** essas mesmas telas permitem **modificar** os cookies — é o que fizemos no lab do [50.md](50.md) usando o Burp Suite. Para edição mais ágil sem proxy, use a extensão **Cookie Editor** (Chrome/Firefox).

## 🧠 Conceitos para Memorizar

| Termo | Definição |
|-------|-----------|
| **Cookie** | Arquivo de texto pequeno guardado pelo navegador |
| **Session Cookie** | Vive na RAM, morre ao fechar o browser |
| **Persistent Cookie** | Vive no disco até a data de expiração |
| **First-party Cookie** | Do próprio site visitado |
| **Third-party Cookie** | De domínios externos embutidos no site |
| **Set-Cookie** | Header HTTP que o servidor usa para criar/atualizar cookies |
| **LGPD** | Lei brasileira de proteção de dados (2018) |
| **Consentimento** | Aceite **expresso, granular e reversível** do usuário |

## 💡 Reflexão Final

> Cookies não são vilões — são uma ferramenta **essencial** para a web funcionar (sem eles, você teria que logar em cada clique).
>
> O problema não é o cookie em si, mas:
> - **Como ele é usado** (rastreamento abusivo, sem consentimento)
> - **Como ele é protegido** (sem `HttpOnly`/`Secure` → roubo de sessão)
> - **Que dado ele carrega** (privilégio editável vs ID opaco)
>
> **Como usuário:** revise suas preferências de cookies periodicamente.
> **Como desenvolvedor:** consentimento granular + cookies opacos + flags de segurança.
