## 📖 Módulo 1 — Arquitetura de Rede

### O que é uma rede?

> Uma **rede de computadores** é um **grupo de dispositivos conectados** que trocam dados e compartilham recursos.

### 🧩 Dispositivos de rede

Cada dispositivo tem **endereço MAC** (físico, único) + **endereço IP** (lógico) e uma **interface de rede** que envia/recebe **pacotes de dados** (que carregam **origem** e **destino**). Os *dispositivos de rede* (roteadores, switches…) gerenciam o que trafega; os *dispositivos finais* (PC, celular) se conectam através deles.

| Dispositivo | Função | Camada TCP/IP | Nota de segurança |
|-------------|--------|---------------|-------------------|
| **🧱 Firewall** | Monitora e **restringe** o tráfego de entrada/saída conforme regras da organização | — | **Primeira linha de defesa**; fica entre a rede interna confiável e a internet (não confiável) |
| **🖥️ Servidor** | Fornece informações/serviços aos **clientes** (modelo cliente-servidor) | — | Ex.: **DNS** (resolve nomes), **arquivos**, **e-mail** corporativo |
| **🔌 Hub** | Ponto comum de conexão que **repete tudo para todas as portas** | Enlace | ⚠️ Vulnerável a **espionagem (eavesdropping)**; pouco usado hoje |
| **🔀 Switch** *(interruptor)* | Encaminha pacotes **só ao destino**, usando uma **tabela de endereços MAC** | **Enlace de dados** | Mais **seguro e eficiente** que o hub — padrão nas redes modernas |
| **🧭 Roteador** | Conecta **redes diferentes** e direciona pelo **IP de destino** (lê o cabeçalho IP) | **Rede** | Pode ter **firewall embutido** |
| **📡 Modem** | Liga a casa/escritório ao **ISP** (converte os sinais da internet) | — | Empresas grandes usam outras tecnologias de banda larga |
| **📶 Ponto de acesso sem fio (WAP)** | Cria rede **sem fio** por ondas de rádio; dispositivos conectam via **Wi-Fi** | — | **Wi-Fi** = conjunto de padrões de comunicação sem fio |

> 🔗 **Modelo cliente-servidor:** o **cliente** pede informação/serviço; o **servidor** executa o pedido (ex.: o navegador pergunta a um **servidor DNS** o IP de um site).
>
> 🔌 **Hub × 🔀 Switch:** o hub faz **broadcast** para todos (espionável); o switch envia **só ao destinatário** — por isso domina as redes modernas.

### 🗺️ Diagramas de rede

São **mapas** que mostram os dispositivos e como se conectam (ícones + linhas pontilhadas). Analistas de segurança os usam para **visualizar a arquitetura** e refinar as estratégias de proteção. Fluxo típico de uma rede doméstica:

```
Internet → Modem (ISP) → Roteador (+Firewall) → Switch → [PCs · laptops · celulares · impressora · servidor de arquivos]
```

### 🌍 Tipos de rede & nuvem

| Termo | O que é |
|-------|---------|
| **🏠 LAN (Local Area Network)** | Rede em **área limitada** (casa, escritório, prédio) |
| **🌐 WAN (Wide Area Network)** | Rede em **grande área geográfica** (ex.: a própria internet) |
| **☁️ Nuvem (cloud)** | Conjunto de **servidores/computadores** que armazena recursos e dados em **data centers remotos**, acessados pela **internet** |
| **🖥️ Ferramenta de virtualização** | Software que cria **recursos virtuais** (ex.: máquinas virtuais) a partir de hardware físico |

> 💡 **LAN × Nuvem:** a LAN é **local e limitada**; a nuvem é **remota** e acessada via internet — são conceitos opostos (boa pegadinha de prova).

### ☁️ Computação em nuvem & SDN

- **Computação em nuvem:** usar servidores, aplicativos e serviços de rede **remotos hospedados na internet**, em vez de infraestrutura física própria (rede tradicional = tudo local).
- **CSP (Cloud Service Provider):** empresa dona de **data centers** com milhões de servidores; vende computação/armazenamento/rede via **API** ou **console web**.

**3 modelos de serviço (SaaS · IaaS · PaaS):**

| Modelo | O que o CSP entrega | Quem usa |
|--------|---------------------|----------|
| **SaaS** (Software as a Service) | **Software pronto** operado pelo CSP, usado remotamente | Usuário final (ex.: Gmail) |
| **IaaS** (Infrastructure as a Service) | **Componentes virtuais** (servidores, armazenamento) configurados via API | Admin de infraestrutura |
| **PaaS** (Platform as a Service) | **Ferramentas** para desenvolver apps personalizados na nuvem | Desenvolvedor |

**Ambientes:**
- **☁️➕🏠 Nuvem híbrida** = CSP **+** infraestrutura local própria (modelo mais comum — reduz custo e mantém controle).
- **☁️☁️ Multi-nuvem (multi-cloud)** = uso de **mais de um CSP**.

**SDN (Software-Defined Networking / rede definida por software):** dispositivos e serviços de rede **virtuais** (switches, roteadores, firewalls virtuais). O roteamento é feito por **software**, permitindo configuração rápida, dinâmica e programável.

**🎯 3 benefícios da nuvem/SDN:**

| Benefício | Por quê |
|-----------|---------|
| **Confiabilidade** | Alta disponibilidade, conexões seguras, mínima interrupção |
| **Custo** | Sem grande investimento inicial — **paga-se pelo uso** |
| **Escalabilidade** | Modelo **elástico**: paga só o que precisa, quando precisa; muda rápido via API |

> 💡 Na nuvem dá pra configurar rapidamente defesas como **WAF** (firewall de aplicação web), **IDS/IPS** e **firewalls L3/L4** sempre que necessário.

> 💡 Pegadinha clássica: "grupo de dispositivos conectados" = **rede de computadores** (não *hub*, que é só um dos dispositivos que conectam).

## ✅ Quiz — Módulo 1 (Resolvido)

**1. Um _____ é um grupo de dispositivos conectados.**
> ✅ **Rede de computadores.** (❌ hub = conecta/repassa; roteador = direciona entre redes; MAC = identificador do dispositivo)

**2. Um _____ é um dispositivo de rede que transmite informações a todos os dispositivos da rede.**
> ✅ **Hub** (broadcast para todos). ❌ switch envia só ao destino; roteador direciona entre redes; modem liga à internet.

**3. (V/F) Para conectar uma cidade inteira, a LAN seria a mais eficaz.**
> ✅ **Falso** — uma cidade exige uma **WAN**; a LAN cobre áreas limitadas.

**4. Garantir que a informação seja transmitida a TODOS os computadores da rede → que dispositivo investigar?**
> ✅ **Hub** (faz broadcast para todos).

**5. Benefícios dos switches? (todos que se aplicam)**
> ✅ Passam os dados **só ao destino** · ✅ **Melhoram o desempenho** da rede · ✅ **Controlam o fluxo de tráfego**. ❌ "instalam software de proteção automaticamente".

**6. Servidores, aplicativos e serviços hospedados na internet = _____ computing.**
> ✅ **Nuvem** (cloud).

## 📦 Módulo 2 — Operações de Rede

### Pacote de dados

> Um **pacote de dados** é a **unidade básica de informação** que viaja de um dispositivo para outro em uma rede. Ele carrega, no **cabeçalho**, os endereços de **origem e destino** + os **dados** em si (payload).

> 🕵️ **Packet sniffing** *(captura de pacotes)*: a prática de **capturar e inspecionar** o tráfego de rede. Útil para diagnóstico/defesa — e também usada por atacantes para **espionar** dados (lembre: o **hub** é vulnerável a isso por fazer broadcast).

### 🧱 Modelo TCP/IP (4 camadas)

Estrutura para visualizar como os dados são organizados e transmitidos — ajuda o analista a identificar **qual camada** foi afetada num ataque. Um **protocolo de rede** é um conjunto de padrões para **rotear e endereçar** pacotes. **Ordem (1 → 4):**

| # | Camada | Função | Protocolos / exemplos |
|---|--------|--------|----------------------|
| 1 | **Acesso à rede** (Network Access / enlace) | Cria os pacotes e os transmite no **hardware físico** | Hubs, modems, cabos, fiação · **ARP** |
| **2** | **Internet** | Garante a entrega ao **host de destino** (rede diferente); anexa o **IP** | **IP**, **ICMP** |
| 3 | **Transporte** (Transport) | Entrega entre sistemas + **controle de fluxo** e **portas** | **TCP**, **UDP** |
| 4 | **Aplicação** (Application) | Faz/responde requisições; serviços que o usuário acessa | **HTTP**, **SMTP**, **SSH**, **FTP**, **DNS** |

> 🧠 Macete da ordem: **A**cesso → **I**nternet → **T**ransporte → **A**plicação. A **2ª camada é a Internet**.

**Protocolos-chave por camada:**

| Protocolo | Camada | Para quê |
|-----------|--------|----------|
| **ARP** (Address Resolution) | Acesso à rede | Mapeia **endereço IP ↔ MAC** na rede local |
| **IP** (Internet Protocol) | Internet | Roteia/endereça pacotes entre redes |
| **ICMP** (Control Message) | Internet | Reporta **erros e status** de pacotes (diagnóstico, ex.: ping; pacotes descartados/perdidos) |
| **TCP** (Transmission Control) | Transporte | Conexão **confiável**; retransmite dados perdidos/corrompidos; usa **porta** de destino |
| **UDP** (User Datagram) | Transporte | **Sem conexão**, sem garantia — rápido, para **tempo real** (streaming de vídeo/voz) |
| **HTTP · SMTP · SSH · FTP · DNS** | Aplicação | Web · e-mail · acesso remoto seguro · arquivos · resolução de nomes |

> ⚖️ **TCP × UDP:** TCP = **confiável** (estabelece conexão, retransmite); UDP = **rápido** (sem conexão, ideal para vídeo/voz em tempo real).
>
> ⚖️ **TCP/IP (4 camadas) × OSI (7 camadas):** o **OSI** é o modelo **detalhado** (7 camadas) usado para comunicar onde está um problema; o **TCP/IP** é a versão **simplificada** (4 camadas) usada na prática. A camada de **Aplicação** do TCP/IP equivale às camadas **Aplicação + Apresentação + Sessão** do OSI. *Este curso segue o TCP/IP.*
