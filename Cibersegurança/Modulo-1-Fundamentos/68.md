## 🎯 Visão Geral do Laboratório

Vamos montar um ambiente realista com **3 máquinas virtuais** rodando em paralelo, simulando uma topologia **DMZ profissional** ([67.md](67.md) explica o conceito).

### Topologia que vamos construir

```
┌──────────────────────────────────────────────────────────────┐
│                    SUA MÁQUINA HOST                          │
│                                                              │
│  ┌────────────────┐                                          │
│  │  🔥 firewall   │ 4 interfaces:                            │
│  │    (VM)        │  ├─ Bridge (acesso à internet real)      │
│  │                │  ├─ Host-only (você acessa do host)      │
│  │                │  ├─ Internal: DMZ                        │
│  │                │  └─ Internal: DMZEXT                     │
│  └─────┬──────────┘                                          │
│        │                                                     │
│   ┌────┴────────────────────┐                                │
│   │                         │                                │
│   ▼ (DMZEXT)                ▼ (DMZ)                          │
│  ┌────────────┐       ┌────────────┐                         │
│  │ 🛡️ WAF     │       │ 🖥️ server  │                         │
│  │    (VM)    │       │    (VM)    │                         │
│  │            │       │            │                         │
│  │ recebe     │       │ hospeda    │                         │
│  │ tráfego    │       │ a app web  │                         │
│  │ HTTP/S     │       │ alvo       │                         │
│  └────────────┘       └────────────┘                         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### O papel de cada VM

| VM | Função | Rede |
|----|--------|------|
| **firewall** | Roteador central com 4 interfaces — separa as zonas | Bridge + Host-only + DMZ + DMZEXT |
| **WAF** | Filtra tráfego HTTP antes de chegar ao server | DMZEXT (zona externa) |
| **server** | Aplicação web a ser protegida | DMZ (zona interna do servidor) |

## 💿 Instalando o VirtualBox

### 🪟 Windows

1. Acesse [virtualbox.org/wiki/Downloads](https://www.virtualbox.org/wiki/Downloads)
2. Clique em **Windows hosts** → baixa um `.exe`
3. Execute o instalador
4. Avance com **Next → Next → Install** (configurações padrão funcionam)
5. Aceite os drivers de rede do Oracle (aparecem em pop-ups)

### 🐧 Linux (Ubuntu/Debian)

1. Acesse [virtualbox.org/wiki/Linux_Downloads](https://www.virtualbox.org/wiki/Linux_Downloads)
2. Baixe o `.deb` da sua distro
3. Abra com gerenciador de pacotes ou via terminal:
   ```bash
   sudo dpkg -i virtualbox-*.deb
   sudo apt-get install -f   # corrige dependências se faltar
   ```
4. VirtualBox aparece nas aplicações

### 🍎 macOS

1. Acesse a página de Downloads → **OS X hosts** → baixa um `.dmg`
2. Abra o `.dmg` → execute `VirtualBox.pkg`
3. **Continuar → Instalar** → digite a senha do Mac
4. ⚠️ **Importante no macOS moderno:** pode aparecer aviso em **Preferências do Sistema → Segurança e Privacidade** pedindo para autorizar a Oracle. Clique em **Permitir**

## 📦 Importando as VMs

Com VirtualBox instalado, repita o processo abaixo para **cada um dos 3 arquivos** (`firewall.ova`, `WAF.ova`, `server.ova`):

### Passo a passo

1. Abra o **VirtualBox**
2. Menu superior → **Arquivo → Importar Appliance** (ou botão **Importar** na tela inicial)
3. Clique no ícone de pasta → selecione o arquivo `.ova` baixado
4. **Próximo** → revise os dados (CPU, RAM, disco — pode manter padrão)
5. **Importar** → aguarde alguns minutos

A VM aparece na lista lateral do VirtualBox quando termina.

> 💡 **Repita 3x** — uma vez para cada arquivo `.ova`.

## 🌐 Configuração de Rede (a parte mais crítica!)

⚠️ **Atenção:** **antes de ligar qualquer VM**, configure a rede de cada uma. Sem isso, a topologia não funciona.

Para cada VM: clique com botão direito → **Configurações → Rede**.

### 🔥 VM: `firewall` — 4 adaptadores

Esta é a VM central. Habilite **4 adaptadores de rede**:

| Adaptador | Tipo | Nome | Função |
|-----------|------|------|--------|
| **1** | Placa em modo **Bridge** | (sua interface física) | Acesso à internet real |
| **2** | Placa de rede **exclusiva de hospedeiro** | (vboxnet0 ou padrão) | Você acessa o firewall do host |
| **3** | **Rede interna** | `DMZ` | Conecta ao server |
| **4** | **Rede interna** | `DMZEXT` | Conecta ao WAF |

### 🖥️ VM: `server` — 1 adaptador

| Adaptador | Tipo | Nome |
|-----------|------|------|
| **1** | **Rede interna** | `DMZ` |

### 🛡️ VM: `WAF` — 1 adaptador

| Adaptador | Tipo | Nome |
|-----------|------|------|
| **1** | **Rede interna** | `DMZEXT` |

## 📋 Resumo Visual da Configuração de Rede

```
┌─────────────┐   Adaptador 1: Bridge → Internet
│  firewall   │   Adaptador 2: Host-only → você acessa
│             │   Adaptador 3: Internal "DMZ" ─────────────┐
│             │   Adaptador 4: Internal "DMZEXT" ──┐       │
└─────────────┘                                    │       │
                                                   │       │
                                                   ▼       ▼
                                            ┌──────────┐ ┌────────┐
                                            │   WAF    │ │ server │
                                            │ DMZEXT   │ │  DMZ   │
                                            └──────────┘ └────────┘
```

### Explicando os 4 tipos de rede do VirtualBox

| Tipo | O que faz |
|------|-----------|
| **Bridge** | A VM ganha um IP real na sua rede física — como se fosse outro PC plugado no roteador |
| **Host-only** | A VM só se comunica com o host (sua máquina física). Útil para gerenciar a VM |
| **Rede interna (named)** | VMs com o **mesmo nome de rede** conversam entre si. Isolada do host e da internet |
| **NAT** | A VM acessa internet via host, mas o host **não acessa** a VM (não usamos aqui) |

> 💡 **Por que `DMZ` e `DMZEXT` são "Rede interna"?** Porque queremos **isolar** essas conexões — só as VMs configuradas com esses nomes podem se comunicar. É o que recria o conceito de DMZ ([67.md](67.md)) em laboratório.

## 🐞 Troubleshooting Comum

### "VM não inicia — erro de VT-x/AMD-V"

**Causa:** virtualização desabilitada na BIOS.
**Solução:** entre na BIOS/UEFI ao ligar o PC → habilite **Intel VT-x** ou **AMD-V** → também habilite **VT-d** se possível.

### "VirtualBox conflita com Hyper-V no Windows"

**Causa:** Windows usa Hyper-V exclusivamente.
**Solução:** abra **PowerShell como admin** e rode:
```powershell
bcdedit /set hypervisorlaunchtype off
```
Reinicie. Para reabilitar Hyper-V depois: `bcdedit /set hypervisorlaunchtype auto`.

### "VMs não enxergam umas às outras"

**Causa:** nome da "Rede interna" diferente entre VMs.
**Solução:** confirme que firewall (Adaptador 3) e server estão com **exatamente** `DMZ` (case-sensitive). E firewall (Adaptador 4) e WAF com **`DMZEXT`**.

### "Bridge não pega IP — fica em 169.254.x.x"

**Causa:** placa de rede selecionada errada (escolheu Wi-Fi quando deveria ser Ethernet, ou vice-versa).
**Solução:** vá em **Configurações → Rede → Adaptador 1** e selecione a **interface real ativa** no seu host.

### "Importação falha — disco corrompido"

**Causa:** download incompleto.
**Solução:** baixe novamente o `.ova` e compare o tamanho do arquivo.

### macOS bloqueando a instalação

**Causa:** Gatekeeper desconfia da Oracle.
**Solução:** **Preferências → Segurança e Privacidade** → libere o app Oracle. Pode precisar reiniciar.

## 🧠 Por Que Esse Setup é Realista

Esse laboratório recria **a arquitetura clássica** que vimos em [67.md](67.md):

```
Internet → Firewall → WAF → Server da aplicação
                        ↑
              (filtragem camada 7 antes do servidor)
```

| No mundo real | No laboratório |
|---------------|----------------|
| Cloudflare/Barracuda na borda | **WAF** (VM) |
| Roteador + iptables/pfSense | **firewall** (VM) |
| Servidor web em DMZ | **server** (VM) |
| Segmentação física via VLAN | "**Rede interna**" no VirtualBox |
| Public Subnet (AWS) | DMZ + DMZEXT |
| Private Subnet (AWS) | (não temos no lab, mas seria outra "Rede interna") |

> 🎯 **A grande sacada:** quando você for trabalhar com firewalls reais (pfSense, Fortinet) ou cloud (AWS Security Groups, Azure NSG), a **mecânica é a mesma** — só muda a interface.
